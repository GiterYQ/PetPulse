# MVP 上线前检查清单

> 当前状态：模板。真实 App、TestFlight、App Store / 应用商店提审前按实际行为更新。

## 1. 产品范围

- [ ] MVP 功能和后置功能已在 `docs/product/MVP功能分层与隐藏功能清单.md` 更新。
- [ ] 用户可见入口没有“点进去才发现不可用”的假功能。
- [ ] AI 功能没有医疗诊断或救援保证表达。

## 2. 隐私与数据

- [ ] 隐私政策覆盖账号、宠物档案、照片、位置、通知 token。
- [ ] 精确位置使用有明确授权和说明。
- [ ] 删除账号、删除宠物、删除求助有设计。
- [ ] 后台不默认暴露用户敏感原文。
- [ ] 匿名统计和用户原始内容分开处理。

## 3. 本地化与区域

- [ ] 用户可见文本已走本地化 key 或有迁移计划。
- [ ] 地图、推送、登录等第三方能力没有写死在业务逻辑里。
- [ ] 中国/海外差异记录在区域策略或 Provider 设计里。
- [ ] 截图和商店文案无不合适地区表达。

## 4. 技术验证

- [ ] 构建通过。
- [ ] 主流程手测通过。
- [ ] 后端健康检查通过。
- [ ] 通知发送链路至少有一次端到端验证。
- [ ] 关键错误有日志或可追踪 request id。

## 5. 商店与审核材料

- [ ] App 名称、简介、截图与实际能力一致。
- [ ] 不写“保证救援”“AI 诊断”“官方医疗认证”等高风险词。
- [ ] Review Notes 说明是否需要登录、如何触发核心功能。
- [ ] 权限说明与实际权限一致。
- [ ] 如有订阅/支付，商品说明不暗示更高救援成功率。
