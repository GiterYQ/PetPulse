# PetPulse 文档组织结构说明

## 当前结论

PetPulse 当前是 React/Vite 展会 Demo + 产品规划文档。未来可能扩展为 Flutter / React Native / Web Admin / 后端服务，但现阶段文档应服务于 MVP 判断和后续开发交接。

## 文档优先级

1. `AGENTS.md`
2. `docs/README.md`
3. `docs/management/文档组织结构说明.md`
4. `docs/management/Markdown文档状态清单_20260615.md`
5. `docs/project-rules.md`
6. `PLAN.md`
7. 现有代码和组件实现

## 目录说明

| 目录 | 作用 |
| --- | --- |
| `docs/management/` | 文档索引、状态清单、同步规则 |
| `docs/engineering/` | 工程规则、AI 协作、本地化、区域化、Provider 抽象 |
| `docs/product/` | 产品范围、隐藏功能、数据同步、合规文案 |
| `docs/qa/` | 手测脚本和专项回归 |
| `docs/release/` | 上线前检查和提审准备 |
| `docs/prompts/` | 项目提示词 |

## 更新规则

- 新增、删除、移动 Markdown 文件时，更新文档状态清单。
- 改动 MVP 范围时，更新产品功能分层文档。
- 改动本地化、地区、Provider 抽象时，更新工程规则。
- 改动发布条件时，更新 QA 或 release 清单。
- 旧结论被新文档吸收后，旧文档应标记为历史参考或删除候选，不要让后续 Agent 误读。
