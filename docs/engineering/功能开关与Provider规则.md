# 功能开关与 Provider 规则

## 1. 原则

PetPulse 的许多能力有地区、合规、合作方和成本差异。MVP 阶段不要把这些差异写死在页面里，应通过功能开关和 Provider 抽象控制。

## 2. 功能开关适用场景

需要开关控制：

- 医院/兽医认证入口。
- 血液捐赠匹配。
- 走失宠物广播范围。
- AI 文案生成。
- AI 匹配解释。
- 云同步。
- 公开联系方式。
- 精确位置展示。
- 支付、订阅、捐款。
- 区域试点功能。

不需要开关控制：

- 基础账号。
- 宠物档案。
- 本地 UI 主题。
- 基础求助状态展示。

## 3. Provider 抽象

建议未来保留这些 Provider：

| Provider | 作用 |
| --- | --- |
| `MapProvider` | 地图显示、地理编码、距离计算 |
| `NotificationProvider` | Push、短信、邮件、站内通知 |
| `AuthProvider` | 邮箱、手机号、Apple、Google、微信等登录 |
| `StorageProvider` | 图片和证明材料存储 |
| `AIProvider` | 文案生成、摘要、翻译、匹配解释 |
| `PaymentProvider` | 订阅、捐款、押金等资金能力，MVP 可暂不接 |
| `RegionPolicyProvider` | 根据地区、渠道、版本返回功能可见性 |

## 4. 隐藏功能记录要求

隐藏功能必须记录：

- 功能名称。
- 当前状态：可见 / 隐藏 / 暂停 / 后置。
- 控制位置：配置、Provider、环境变量或代码位置。
- 隐藏原因。
- 恢复条件。
- 恢复后需要验证什么。

记录位置：`docs/product/MVP功能分层与隐藏功能清单.md`。

## 5. 删除与保留边界

可以删除：

- 未上线旧 Demo 的无用 mock。
- 无引用、无计划、无说明的实验文件。
- 明确被新方案替代的历史文档。

默认保留：

- 网络 fallback。
- Provider fallback。
- 本地化 fallback。
- 用户数据恢复路径。
- 运行时错误提示。
- 配置降级路径。
