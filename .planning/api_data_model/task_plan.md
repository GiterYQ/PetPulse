# PetPulse API 与数据模型草案计划

## 目标

创建一份中文 API 与数据模型草案，承接 `docs/product/MVP需求草案.md` 中的实体清单，明确 PetPulse MVP 后端需要支持的接口、核心数据表、状态流转、隐私边界和多端/多地区预留点。

## 阶段

1. 读取 MVP、Provider、本地化、数据同步和合规文档：完成
2. 检查是否已有重复 API 草案：完成
3. 编写 API 与数据模型草案：完成
4. 更新文档索引和状态清单：完成
5. 复查文档质量：完成

## 约束

- 文档使用中文，API 名、字段名和状态码保留英文。
- 只定义 MVP 后端契约草案，不做代码实现和数据库迁移。
- 后端是长期数据源，客户端可以未来替换为 Flutter、React Native、Web 或原生。
- 不引入支付、保险、复杂认证、AI 医疗诊断和公开精确位置。
