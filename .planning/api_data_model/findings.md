# PetPulse API 与数据模型草案发现

## 已确认口径

- 当前项目是 React/Vite 展会 Demo，没有真实后端。
- MVP 核心闭环是：账号 -> 宠物档案 -> 发起求助 -> 附近匹配 -> 发送提醒 -> 对方响应 -> 历史/信任沉淀。
- 用户、宠物、求助、响应、通知、媒体元数据应归后端保存，客户端只做入口和缓存。
- 中国/海外差异先通过字段、Provider、配置和功能开关处理，不复制两套 schema。
- 精确位置默认不公开，只在求助处理必要范围内保存快照；MVP 不保存长期位置轨迹。
- AI 只用于文案整理、摘要、翻译和匹配解释，不做医疗诊断或救援保证。
